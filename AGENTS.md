# Codex Working Rules

## Highest Priority Encoding Rule

- For every file operation in this repository, including reading, creating, editing, and deleting files, Codex must use UTF-8 encoding.
- When using PowerShell, prefer explicit UTF-8 handling such as `Get-Content -Encoding UTF8`, `Set-Content -Encoding UTF8`, or equivalent UTF-8-safe commands.
- This UTF-8 rule must be preserved in handoff notes, summaries, and future task summaries.
- Do not omit this rule when summarizing the project state or Codex operating rules.
