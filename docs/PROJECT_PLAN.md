# 프로젝트 계획

## 목적

이 프로젝트의 목적은 개인 재무 데이터를 단순 가계부가 아니라 운영 가능한 ERP 관점으로 다루는 것입니다.
핵심은 실제 거래, 반복규칙, 보험, 차량비, 월말 예측을 한 구조 안에서 연결하는 것입니다.

## 완료한 기반 단계

1. 툴링과 CI 정리
2. 인증과 사용자 경계 정리
3. 공용 계약 계층 도입
4. 앱 단위 env 구조 정리
5. migration-first DB 흐름 정리
6. API 레이어 분리
7. 프론트 feature 구조 전환
8. fallback 정책 명시화
9. 최소 테스트 매트릭스 도입
10. 협업용 문서 체계 정리

## 현재 MVP 범위

- 로그인
- 계좌/카테고리 조회
- 거래 조회/생성
- 반복규칙 조회/생성
- 보험 조회
- 차량 조회
- 대시보드 요약
- 월별 예측
- 디자인 시스템 샘플 페이지

## 다음 우선순위

1. 요청 단위 API 통합 테스트 추가
2. Web 로그인 상태와 토큰 주입 연결
3. 거래/반복규칙 폼의 실제 mutation 연결
4. 보험/차량 CRUD 확대
5. 문서와 Swagger 동기화 자동 점검

## 범위 밖 항목

현재 스타터 범위에는 아래 항목을 포함하지 않습니다.

- 외부 금융기관 자동 연동
- AI 기반 재무 조언
- OCR 영수증 처리
- 고급 회계 마감 프로세스
- 운영용 멀티테넌시 관리 UI
