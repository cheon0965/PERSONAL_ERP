# 데모 체험 가이드

## 이 문서의 목표

이 문서는 데모 계정으로 PERSONAL_ERP를 직접 눌러보며 `월별 재무 운영 한 바퀴`를 이해하기 위한 안내서입니다.
상세한 도메인 규칙을 모두 설명하기보다, 데모 화면에서 무엇을 보고 어떤 순서로 움직이면 되는지에 집중합니다.

더 깊은 기준이 필요하면 아래 문서를 함께 보면 됩니다.

- 실행 방법과 현재 범위: `README.md`, `docs/CURRENT_CAPABILITIES.md`
- 도메인 기준: `docs/domain/business-logic-draft.md`, `docs/domain/core-entity-definition.md`
- 운영 점검: `docs/OPERATIONS_CHECKLIST.md`

## 한눈에 보는 데모 흐름

`로그인 -> 현재 사업장 / 장부 확인 -> 운영 허브 점검 -> 기준 데이터 확인 -> 현재 운영월 열기 -> 계획 항목 생성 -> 수집 거래 확인 -> 전표 확정 -> 월 마감 -> 재무제표 생성 -> 차기 이월 -> 기간 전망`

데모에서 기억할 핵심은 하나입니다.
거래 한 건을 입력하는 앱이 아니라, `계획 -> 실제 거래 수집 -> 전표 확정 -> 월 마감 -> 공식 보고 -> 다음 달 준비`까지 이어지는 월 운영 시스템입니다.

## 체험 전 준비

로컬에서 체험한다면 먼저 개발 환경을 실행합니다.

```bash
npm run db:up
npm run db:migrate
npm run db:seed
npm run dev
```

- Web: `http://localhost:3000`
- API: `http://localhost:4000/api`
- Swagger: `http://localhost:4000/api/docs` (`SWAGGER_ENABLED=true`일 때)

데모 계정은 아래 값을 사용합니다.

| 항목     | 값                 |
| -------- | ------------------ |
| 이메일   | `demo@example.com` |
| 비밀번호 | `Demo1234!`        |

`npm run db:seed`는 로컬 개발 DB를 데모 데이터 기준으로 다시 채웁니다.
운영 DB에서는 실행하지 않는 명령입니다.

## 데모 데이터에 준비된 것

시드 데이터에는 데모를 빠르게 시작할 수 있도록 기본 재료가 들어 있습니다.

- 기본 사업장, 기본 장부, 데모 사용자의 OWNER 권한
- `사업 운영 통장`, `비용 예비 통장`, `사업용 카드`
- 수입/지출 카테고리와 system-managed 계정과목/거래유형
- 현재 데모 기준월에 걸리는 반복 규칙
- 보험 계약, 부채 계약과 상환 일정
- 차량 기본 정보, 연료/정비 운영 기록

따라서 첫 체험에서는 기준 데이터를 새로 만드는 데 시간을 쓰기보다, 이미 준비된 기준을 확인하고 월 운영 흐름으로 넘어가는 편이 좋습니다.

## 추천 데모: 10분 안에 월 운영 한 바퀴

아래 순서대로 따라가면 전체 제품 흐름을 가장 빠르게 체험할 수 있습니다.
이미 해당 데이터가 만들어져 있거나 월이 열려 있다면, 새로 만들기보다 상태만 확인하고 다음 단계로 넘어가면 됩니다.

| 순서 | 화면 / 경로                           | 할 일                                                         | 성공 신호                                             |
| ---- | ------------------------------------- | ------------------------------------------------------------- | ----------------------------------------------------- |
| 1    | 로그인 `/login`                       | `demo@example.com` / `Demo1234!`로 로그인합니다.              | 로그인 후 대시보드 또는 보호 화면으로 이동합니다.     |
| 2    | 월 운영 대시보드 `/dashboard`         | 현재 월 운영 요약을 가볍게 봅니다.                            | 현재 사업장 기준의 요약 숫자와 안내가 보입니다.       |
| 3    | 현재 사업장 / 장부 `/settings`        | 현재 사업장, 장부, 내 권한을 확인합니다.                      | 데모 계정이 현재 사업장의 OWNER로 보입니다.           |
| 4    | 운영 허브 `/operations`               | 오늘 우선 확인, 마감 차단, 업로드 상태, 알림 진입점을 봅니다. | 운영상 막힌 작업이 있는지 한 화면에서 감을 잡습니다.  |
| 5    | 기준 데이터 `/reference-data`         | readiness, 자금수단, 카테고리를 확인합니다.                   | 자금수단과 카테고리가 준비된 상태입니다.              |
| 6    | 월 운영 시작 `/periods/open`          | 열린 월이 없으면 현재 운영월을 엽니다.                        | 현재 운영월이 최신 진행월로 잡힙니다.                 |
| 7    | 계획 항목 생성 `/plan-items/generate` | 현재 월 기준으로 계획 항목을 생성합니다.                      | 반복 규칙과 상환 일정에서 계획 항목이 만들어집니다.   |
| 8    | 계획 항목 `/plan-items`               | 생성된 계획과 연결 수집 거래 상태를 확인합니다.               | `전표 준비` 상태의 연결 거래가 보입니다.              |
| 9    | 수집 거래 `/transactions`             | 전표 준비 거래를 검토하고 전표로 확정합니다.                  | 전표 번호가 연결되고 거래 상태가 확정됩니다.          |
| 10   | 전표 조회 `/journal-entries`          | 방금 생성된 전표를 열어 라인과 금액을 봅니다.                 | 공식 회계 기록으로 확정된 결과를 확인합니다.          |
| 11   | 월 마감 `/periods/close`              | 미확정 거래가 없는지 확인하고 현재 운영월을 마감합니다.       | 기간 상태가 잠금 또는 마감 상태로 바뀝니다.           |
| 12   | 재무제표 `/financial-statements`      | 잠금된 현재 운영월 기준 재무제표를 생성합니다.                | 공식 보고 스냅샷이 생성되고 상세 화면에서 확인됩니다. |
| 13   | 차기 이월 `/carry-forwards`           | 마감 결과를 다음 월 오프닝 기준으로 넘깁니다.                 | 다음 달 기초 기준이 생성됩니다.                       |
| 14   | 기간 전망 `/forecast`                 | 현재 월 결과와 다음 달 전망을 확인합니다.                     | 운영 판단용 전망 read model이 이어집니다.             |

첫 월을 여는 화면에서 오프닝 잔액이 필요하다면 예시로 아래처럼 입력하면 됩니다.

| 항목     | 예시             |
| -------- | ---------------- |
| 계정과목 | `현금및예금`     |
| 자금수단 | `사업 운영 통장` |
| 금액     | `2,450,000`      |

## 화면을 읽는 간단한 기준

데모 중 화면이 많아 보여도 역할은 단순합니다.

| 화면 묶음                      | 의미                                                                        |
| ------------------------------ | --------------------------------------------------------------------------- |
| 현재 사업장 / 장부             | 지금 내가 어느 사업장, 어느 장부, 어떤 권한으로 보고 있는지 확인합니다.     |
| 운영 허브                      | 마감 차단, 예외, 업로드 상태, 알림처럼 운영자가 먼저 볼 신호를 모읍니다.    |
| 기준 데이터                    | 자금수단, 카테고리, 계정과목, 거래유형처럼 다른 화면이 참조하는 기준입니다. |
| 반복 규칙 / 계획 항목          | 이번 달에 예정된 거래를 만듭니다. 아직 공식 회계 기록은 아닙니다.           |
| 수집 거래 / 업로드 배치        | 실제 발생했거나 파일에서 읽은 거래 후보를 모으고 검토합니다.                |
| 전표 조회                      | 수집 거래가 공식 회계 기록으로 확정된 결과입니다.                           |
| 월 마감 / 재무제표 / 차기 이월 | 확정된 전표를 기준으로 월을 닫고 보고서와 다음 달 기준을 만듭니다.          |
| 기간 전망                      | 공식 보고서는 아니지만, 다음 운영 판단을 돕는 예측 화면입니다.              |

## 선택 체험 1: 직접 거래 입력

계획 항목 없이 수기로 거래를 넣어보고 싶다면 이 경로를 사용합니다.

`기준 데이터 확인 -> 월 운영 시작 -> 수집 거래 등록 -> 전표 확정 -> 전표 조회 -> 월 마감`

1. `/transactions`에서 새 수집 거래를 등록합니다.
2. 날짜는 화면에 표시된 열린 운영월 안의 날짜를 사용합니다.
3. 자금수단과 카테고리를 선택하고 저장합니다.
4. 저장된 거래가 `전표 준비` 상태가 되면 전표로 확정합니다.
5. `/journal-entries`에서 생성된 전표를 확인합니다.

이 경로는 제품의 최소 회계 흐름을 이해하기 좋습니다.
반복 규칙과 계획 자동화가 없어도 `수집 거래 -> 전표 -> 마감` 구조는 동일합니다.

## 업로드 배치 데모 루트

파일이나 붙여넣기 원본이 수집 거래로 바뀌는 흐름을 보고 싶다면 아래 순서로 확인합니다.

`업로드 배치 -> 행 검토 -> 수집 거래 등록 -> 수집 거래 확인 -> 전표 확정`

1. `/imports`에서 업로드 배치 등록 화면을 엽니다.
2. 원본 형식으로 `직접 붙여넣기`, `계좌 CSV`, `카드 엑셀`, `IM뱅크 PDF` 중 하나를 선택합니다.
3. 텍스트 붙여넣기라면 UTF-8 본문으로 아래와 같은 형식을 사용합니다. `YYYY-MM`은 화면에 표시된 열린 운영월로 바꿔 입력합니다.

```csv
date,title,amount
YYYY-MM-12,사무용품 구입,-33000
YYYY-MM-15,고객 입금,550000
```

4. 배치를 만든 뒤 `/imports/[batchId]` 작업대에서 파싱된 행을 확인합니다.
5. 행별 preview에서 자금수단, 카테고리, 중복 후보, 등록 가능 여부를 봅니다.
6. 필요한 행을 선택해 단건 등록 또는 일괄 등록을 실행합니다.
7. 등록된 행은 `/transactions`에서 수집 거래로 이어집니다.
8. 거래가 `전표 준비` 상태라면 전표 확정까지 진행합니다.

IM뱅크 PDF는 텍스트 레이어가 있는 거래내역 PDF만 지원합니다.
스캔 이미지 PDF는 현재 OCR 범위가 아니므로 업로드 전에 차단됩니다.
업로드 원본은 곧바로 공식 회계 기록이 아니라, 검토 후 수집 거래로 승격되는 후보입니다.

## 선택 체험 2: 운영 자산 둘러보기

월 운영 전에 어떤 기준 데이터가 계획과 거래를 설명하는지 보고 싶다면 아래 화면을 가볍게 둘러보면 됩니다.

| 화면                     | 볼 것                                              |
| ------------------------ | -------------------------------------------------- |
| 보험 계약 `/insurances`  | 보험료, 보장, 만기 같은 반복 비용 기준             |
| 부채 관리 `/liabilities` | 대출 계약과 월별 상환 일정                         |
| 차량 운영 `/vehicles`    | 차량 기본 정보, 연료 기록, 정비 이력               |
| 반복 규칙 `/recurring`   | 월세, 구독료, 보험료처럼 계획 항목으로 펼쳐질 규칙 |

이 화면들은 전표를 직접 확정하는 곳이 아닙니다.
계획 항목과 수집 거래를 더 정확하게 만들기 위한 운영 기준입니다.

## 선택 체험 3: 운영/관리 화면 확인

관리와 운영 지원 범위를 보고 싶다면 아래 화면을 확인합니다.

| 화면                     | 용도                                             |
| ------------------------ | ------------------------------------------------ |
| `/admin/members`         | 현재 사업장 멤버와 역할 관리                     |
| `/admin/navigation`      | DB 기반 사이드바 메뉴와 역할별 노출 관리         |
| `/admin/logs`            | 권한 변경, 초대, 거부 이벤트 같은 감사 로그 확인 |
| `/operations/checklist`  | 운영 체크리스트 확인                             |
| `/operations/exceptions` | 마감이나 데이터 처리 중 확인이 필요한 예외 확인  |
| `/operations/exports`    | UTF-8 CSV 수동 반출                              |
| `/operations/notes`      | 운영 메모와 인수인계 기록                        |

전체 관리자 계정으로 로그인한 경우에는 `/admin/users`, `/admin/tenants`, `/admin/support-context`, `/admin/security-threats`, `/admin/operations`에서 플랫폼 단위 관리 화면도 확인할 수 있습니다.

## 데모 후 기억할 핵심 구조

가볍게는 아래 순서만 기억하면 됩니다.

1. `반복 규칙`은 계획의 원천입니다.
2. `계획 항목`은 이번 달 예정 거래입니다.
3. `수집 거래`는 실제 발생했거나 업로드에서 승격된 거래입니다.
4. `전표`는 공식 회계 확정 결과입니다.
5. `월 마감`은 더 이상 그 달을 마음대로 바꾸지 않겠다는 잠금입니다.
6. `재무제표`와 `차기 이월`은 마감 이후의 공식 산출물입니다.
7. `기간 전망`은 다음 달 운영 판단을 돕는 보조 화면입니다.

즉, PERSONAL_ERP의 데모는 `월을 열고`, `예정과 실제 거래를 모으고`, `전표로 확정하고`, `월을 닫고`, `보고와 다음 달 기준까지 이어지는 흐름`을 체험하는 것이 목적입니다.
