# 검증 메모

## 현재 통과 기준

- `npm run check:quick`
- `npm run test`

## 현재 테스트 범위

API:

- 인증 로그인 성공/실패
- 거래 생성 소유권 검증
- 반복규칙 생성 소유권 검증
- 대시보드 요약 계산
- 월별 예측 계산

Web:

- env 파싱
- demo fallback 활성화/비활성화 정책
- 요청 실패 메시지 안내

## 현재 남은 공백

- 컨트롤러 요청 단위 통합 테스트
- JWT 가드와 DTO validation을 함께 검증하는 e2e 수준 테스트
- 실제 폼 mutation과 라우트 연결 검증
- build 산출물 기준 스모크 검증 자동화

## 해석

현재 검증 체계는 “핵심 회귀를 빠르게 막는 최소 보호선”까지는 올라온 상태입니다.
다만 배선 문제까지 자동으로 막으려면 요청 단위 테스트를 추가해야 합니다.
